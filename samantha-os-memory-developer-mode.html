1. Qué estamos construyendo realmente1. What we're actually building

Samantha no vive en una pestaña del navegador. Vive en el panel del sistema, como un indicador nativo de elementary. La UI que ves se compila como libsam.so, un plugin de Wingpanel, mientras que la inteligencia y el enrutamiento de providers viven en un gateway local llamado elementary-claw.

Samantha doesn't live in a browser tab. It lives in the system panel, as a native elementary indicator. The UI you see is compiled as libsam.so, a Wingpanel plugin, while intelligence and provider routing live in a local gateway called elementary-claw.

2. Cómo funciona su memoria2. How its memory works

La memoria de Samantha no es “magia del modelo”. Es un sistema explícito de archivos dentro del workspace de OpenClaw. Hay dos niveles:

Samantha's memory isn't “model magic”. It's an explicit file-based system inside the OpenClaw workspace. There are two levels:

Rutas de memoriaMemory paths

~/.openclaw/workspace/
├── MEMORY.md
└── memory/
    ├── 2026-04-29.md
    └── 2026-04-28.md

MEMORY.md guarda hechos curados y duraderos. Los archivos diarios en memory/YYYY-MM-DD.md guardan contexto reciente y notas operativas.

MEMORY.md stores curated long-term facts. Daily files in memory/YYYY-MM-DD.md store recent context and operational notes.

3. Modo programador: abrir el entorno real3. Developer mode: opening the real environment

Otra capability clave fue enseñarle a Samantha a entrar en “modo programador”. La idea es simple: si le dices que active ese modo en una carpeta, debe abrir el navegador en localhost:3000, Terminal, el editor nativo de elementary y Files, ya posicionados en la ruta correcta.

Another key capability was teaching Samantha to enter “developer mode”. The idea is simple: if you ask it to activate that mode in a folder, it should open the browser on localhost:3000, Terminal, the native elementary editor, and Files, all already positioned in the right path.

“Activa el modo programador en /home/oscarcode91/elementary-claw-vm
y abre localhost 3000”

En la VM lo validamos de punta a punta. Samantha terminó abriendo epiphany, io.elementary.terminal, io.elementary.code y io.elementary.files. Lo importante aquí es que no estamos fingiendo tool calling: estamos usando herramientas para mover el sistema de verdad.

We validated it end-to-end in the VM. Samantha ended up opening epiphany, io.elementary.terminal, io.elementary.code, and io.elementary.files. What's important here is that we're not faking tool calling: we're using tools to move the real system.

Detalle bonito de elementaryA nice elementary detail

Al principio el fallback buscaba code o codium, pero en la VM el editor correcto era io.elementary.code. Ese tipo de detalles son justo los que solo aparecen cuando pruebas dentro del sistema real.

At first the fallback looked for code or codium, but in the VM the correct editor was io.elementary.code. That's exactly the kind of detail that only appears when you test inside the real system.

4. Cuando GitHub falla, Samantha no debería verse rota4. When GitHub fails, Samantha shouldn't look broken

Uno de los problemas más molestos fue este mensaje crudo de GitHub Copilot: “Sorry, you've exceeded your 5 hour session limits…”. Eso no se le debe enseñar al usuario final como si fuera la UX oficial del asistente.

One of the most annoying problems was GitHub Copilot's raw message: “Sorry, you've exceeded your 5 hour session limits…”. That should not be shown to the end user as if it were the assistant's official UX.

La solución fue interceptar ese fallo en el gateway, clasificarlo y hacer que el panel ofrezca un flujo alternativo. Primero intentamos con API key, pero el flujo correcto para el producto terminó siendo suscripción ChatGPT/Codex como opción principal y API key como fallback secundario.

The solution was to intercept that failure in the gateway, classify it, and make the panel offer an alternative flow. We first tried an API key flow, but the right product choice ended up being ChatGPT/Codex subscription as the primary option and API key as a secondary fallback.

5. La VM importa más de lo que parece5. The VM matters more than it seems

Muchas cosas “se ven bien” en local hasta que las corres en una VM ARM64 con elementary OS de verdad. Ahí fue donde vimos si Wingpanel cargaba el plugin correcto, si el servicio levantaba, si el editor nativo existía, y si la memoria estaba cayendo en el path correcto.

A lot of things “look good” locally until you run them inside a real ARM64 elementary OS VM. That's where we saw whether Wingpanel loaded the correct plugin, whether the service booted, whether the native editor existed, and whether memory landed in the correct path.

Lección claveKey lesson

Un agente de escritorio no se valida solo con tests unitarios. Necesita pruebas dentro del sistema operativo porque la mitad de los bugs viven en la fricción entre UI, procesos, rutas, permisos y apps instaladas.

A desktop agent can't be validated with unit tests alone. It needs OS-level testing because half the bugs live in the friction between UI, processes, paths, permissions, and installed apps.
